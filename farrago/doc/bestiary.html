Farrago Bestiary

Here's a quick rundown on some of the project-specific naming conventions you may encounter while digging through the Farrago codebase.

Prefixes

Most public Java classes follow a naming scheme to help you quickly tell which subsystem they fall under. Here's a list of the most common prefixes (TODO: links):
  • Fennel: the C++ storage manager. Java classes with this prefix either participate in the wrapper JNI layer (e.g. FennelStorage) or represent Fennel-specific aspects of storage/transaction management and query processing (e.g. FennelIndexScanRel).
  • Eigenbase: the organization behind Farrago. This is also used as a package name for components which are shared across the various Eigenbase projects.
  • OJ: OpenJava, the Java language object model used by the code generation framework.
  • Volcano: the generic optimizer, which Farrago specializes with its own transformation rules.
  • Rex: SQL row expressions
  • Mdr: the generic metadata repository used to store and access the system catalog.
  • Cwm: the Common Warehouse Metamodel. This is the basis for the system catalog. Most classes with this prefix (e.g. CwmTable) are generated by the build from the CWM specification. However, Farrago also provides some of its own non-generated specializations (e.g. CwmTableImpl).
  • Fem: the Farrago Extension Model. This is the Farrago-specific portion of the system catalog. Most classes with this prefix (e.g. FemDbHandle) are generated by the build from a UML model defined by the Farrago project.
  • FarragoJdbc: the Farrago JDBC driver.
  • FarragoJdbcEngine: the engine/server side of the Farrago JDBC driver.
  • FarragoJdbcClient: the client side of the Farrago JDBC driver.
  • RJ, RmiJdbc: the RMI package used to connect the Farrago JDBC client with the Farrago server.
  • Med: SQL/MED, the SQL standard extension for management of external data.
  • FarragoMed: Farrago's non-standard SPI for SQL/MED wrappers.
  • MedJdbc: SQL/MED implementation for access to foreign JDBC data.
  • MedMdr: SQL/MED implementation for access to metadata in any MDR repository.
  • Ftrs: Fennel transactional row store; the BTree-based reference implementation used to store local tables by default.
  • Rng: random number generator; this is used as an example of how to plug new functionality into Farrago.
  • Lurql: a metadata query language used internally by Farrago.
Other more common prefixes are also used (e.g. Ddl for data definition language). Classes with the Farrago prefix (e.g. FarragoDatabase) are not specific to any one subsystem.
