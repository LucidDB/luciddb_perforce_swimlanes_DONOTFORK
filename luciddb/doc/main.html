LucidDB is the first and only open-source RDBMS purpose-built entirely for data warehousing and business intelligence. It is based on architectural cornerstones such as column-store, bitmap indexing, hash join/aggregation, and page-level multiversioning. Most database systems (both proprietary and open-source) start life with a focus on transaction processing capabilities, then get analytical capabilities bolted on as an afterthought (if at all). By contrast, every component of LucidDB was designed with the requirements of flexible, high-performance data integration and sophisticated query processing in mind. Moreover, comprehensiveness within the focused scope of its architecture means simplicity for the user: no DBA required.

Rather than throwing hardware at data warehousing problems by relying on expensive clusters or specialized "appliances", the scalability offered by LucidDB's unique architecture allows you to achieve great performance using only a single off-the-shelf Linux or Windows server. Besides keeping costs down, this also minimizes maintenance and administration hassles.

Project News

24-May-2009: Release 0.9.1 is now available. This is a maintenance release with a few critical bugfixes.

22-Apr-2009: Release 0.9.0 is now available. Major new features include UNICODE support, ALTER TABLE ADD COLUMN, UPDATE, and related upsert optimizations. John Sichi is giving a LucidDB talk at the Percona Performance Conference on April 23.

13-Dec-2008: A Pentaho Data Integration plugin for LucidDB bulk load is now available.

12-Nov-2008: Release 0.8.0 is now available. Major new features include warehouse labels and hot/incremental backup.

19-Oct-2008: An automatic Mondrian aggregate table designer add-on for the 0.7.4 release is now available.

16-Jul-2008: Release 0.7.4 is now available. This is primarily a maintenance release, but new features include support for constant reduction of uncorrelated subqueries by the optimizer, as well as a number of new UDX's. Benchmark results are being presented at OSCON 2008.

22-Mar-2008: Release 0.7.3 is now available. The major new feature in this release is intelligent prefetch for all access paths; together with improvements in data caching and asynchronous I/O, these provide improved performance when accessing large amounts of data on disk.

17-Dec-2007: We're looking for LucidDB users who are able to act as references for a report on open-source data warehousing. If you are using LucidDB and can share your experience, please let us know at the users mailing list.

23-Nov-2007: Release 0.7.2 is now available. The major new feature in this release is the ESTIMATE STATISTICS clause for faster ANALYZE TABLE. The TABLESAMPLE query construct is provided as a companion feature.

18-Aug-2007: Release 0.7.1 is now available. Major new features include nested loop join support (used for implementing arbitrary outer join conditions) and Mondrian source replication.

8-Jun-2007: Release 0.7.0 is now available, with support for a number of major new features:

  • read/write concurrency with snapshot consistency
  • cost-based index selection
  • 64-bit Linux binaries available via Sourceforge download
